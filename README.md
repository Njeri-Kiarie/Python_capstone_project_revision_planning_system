# Soma Planner
## Project Description
Soma Planner is a revision application that is designed to help students optimize, manage and schedule their study time especially during the exam period. By inputting their exam dates, subjects and difficulty weights, the system will calculate a personalized study timetable and generate a study plan based on content difficulty.

## Problem Statement
Students struggle to allocate the limited exam preparation time efficiently especially when they have subjects that they are struggling with as they juggle with the competing deadlines.

## Target Audience
University students preparing for multi-subject exam periods.

## Planned Features
1. Subject management: Adding, Viewing, Updating and Deleting Subjects.
2. Priority-based study planning by using a priority calculator formula : priority = subject.difficulty / (10*days_remaining). The value "10" has been added for priotiy readability and does not alter with priority ranking.
3. Automatic study plan generation.
4. Study plan export to csv file.
6. Persistent Data storage in the soma database.
7. Dynamic sorting of revision priorites.

## Technologies Used
1. Python: OOP, Exception Handling, File handling, Date and time manipulation and Randomization.
2. Database technology: SQLite for database tables such as: SUBJECTS TABLE to store all the subjects the user wants to study, EXAMS TABLE to store exam dates for each subject and NOTES TABLE that will link the user study notes to a subject.
3. Algorithms: Study priority that will calculate revision priority based on subject difficulty and days remaining until the exam, Sorting which will sort the exams from highest to lowest priority.
4. Data Structures: Dictionaries, Lists and Tuples.
5. File Handling: Open, Write and Close files using with for csv files to export study plans.
6. Exception Handling: try-except to handle invalid input, invalid dates, database errors and file errors.
7. Database Abstraction: db.execute()

## Data Source
Soma Planner primarily relies on user-provided data including subject information and exam schedules for creating personalized study plan.

## Success Criteria
The project will be considered successful if students can register the subjects with regard to scheduled exams, generate prioritized study plans based on exam date allocation and subject difficulty. The system should also provide useful feedback and handle invalid inputs gracefully.
   
## Stretch Goals
* Email or Notification reminders when exams are approaching.
* Automatic Quiz generation.
* Calendar Integration.
* Mobile or Web version.

Given more development time, the system could be extended with user authentication, quiz history and analytics, calendar integration and reminder notifications. Currently, the system generates a priority based study planner only. A future enhancement would be an autogenerated quiz system that allows the user to upload their notes and questions/flashcards generated to enhance their revision.
